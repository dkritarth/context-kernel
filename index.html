context-kernel

Personal LLM context memory, self-hosted and under your control.

LIVE https://context-kernel.dkritarth.workers.dev/mcp

context-kernel answers one question: what context should an authorized Claude session know about me right now? You curate a set of Markdown files about yourself and your work. A Cloudflare Worker serves them to Claude (Claude Code, Desktop, and browser chat) over a single authenticated remote MCP connector. Your agents can extend the memory through an append-only journal, but only you promote journal notes into the curated source of truth.

It is not a personal website, resume renderer, CMS, or chatbot. No public frontend, no unauthenticated access.

Built with

  • Models: Claude Sonnet 5 (main build), Claude Haiku 4.5 (final polish). 170k+ tokens across implementation, testing, deployment.
  • Runtime: Cloudflare Workers (V8 isolate), minimal deps (agents SDK, MCP TypeScript SDK, Zod).
  • Storage: Cloudflare KV (curated context + journal buffer, separate namespaces).
  • Language: TypeScript. Strict mode, comprehensive tests via vitest.
  • Protocol: Remote MCP over HTTP with bearer token authentication.

Architecture

content/*.md  (hand-edited source of truth)
      |
      v
  scripts/generate-artifacts.ts  (build KV bulk payload)
      |
      v
  Cloudflare KV (CONTEXT_KV namespace)
      |
      v
  src/worker.ts (MCP server handler)
      |
      +-- src/mcp/auth.ts (constant-time bearer validation)
      +-- src/mcp/tools.ts (MCP tool handlers)
      +-- src/kv.ts (read-only KV helpers)
      |
      v
  MCP endpoints (read: get_context, list_sections, get_meta)
                 (write: append_journal, list_journal)
                 |
                 +-- Authentication: two tokens, two scopes (read/write)
                 +-- Journal: append-only, ULID keys, manual promotion gate

Key Design Decisions

  • Hand-curated Markdown, not vector search. Nothing becomes memory without explicit human approval. Avoids the semantic-drift problem of vector-store memory when facts change over time.
  • Append-only journal, manual promotion. Agents can leave notes but cannot corrupt curated context. The promotion step is where the owner filters signal from noise.
  • Two token scopes, separate secrets. Read token for sessions, write token for servers. Write leak is contained (notes only), read leak is the primary risk (context exposure).
  • Constant-time token comparison. No timing attacks. Uses crypto.subtle.digest (SHA-256 + XOR accumulate), portable across workerd and Node.

How to use it

Claude Code CLI (Today, Works)

claude mcp add --transport http context-kernel \\
  https://context-kernel.dkritarth.workers.dev/mcp \\
  --header "Authorization: Bearer <READ_TOKEN>"

The skill skill/context-kernel/SKILL.md is auto-enabled and pulls your context on every session start.

Scripts and Agents (Today, Works)

Local servers, scheduled tasks, and agent workflows can use the write token to append journal entries. The scripts/promote.ts CLI helps you review and promote those notes into curated content by hand.

claude.ai Chat (Not Yet)

Would need OAuth support (currently blocked by provider library incompatibility). Claude Code CLI works as a workaround.
