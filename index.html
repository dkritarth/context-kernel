🧠 context-kernel

Self-hostable LLM context memory. Keep your context, preferences, and knowledge in sync across Claude—without re-pasting or semantic drift.

Self-hosted on Cloudflare Deploy your own • Control your data
The Problem: Running agentic sessions across machines? Stop re-pasting who you are, your goals, writing style, and project status. context-kernel keeps it in one place.

You curate a set of Markdown files about yourself and your work. A Cloudflare Worker serves them to Claude (Claude Code, Desktop, chat) via authenticated remote MCP. Your agents extend the memory through an append-only journal—but only you decide what becomes permanent. The curation gate prevents the memory from rotting.

Not: a website, resume renderer, CMS, or chatbot. No public access, no unauthenticated reads.

Why Not Vector-Memory Tools?

context-kernel

  • Hand-curated Markdown
  • Append-only journal
  • Manual promotion gate
  • Stale content removed by you
  • Trustworthy (you control it)

Vector-Memory (mem0, etc.)

  • Auto-extracted facts
  • Often direct edits
  • Automatic ranking
  • Relies on decay heuristics
  • Semantic drift risk

The gap: Vector tools excel at semantic search but have a known failure mode. When facts change (e.g., "Prod: Postgres 14" → "Postgres 16"), both sit in the index. Similarity search hands back whichever scores higher—usually the older, reinforced one. Result: outdated info looks authoritative.

Architecture

content/*.md  (hand-edited source of truth)
      |
      v
  scripts/generate-artifacts.ts  (build KV bulk payload)
      |
      v
  Cloudflare KV (CONTEXT_KV namespace)
      |
      v
  src/worker.ts (MCP server handler)
      |
      +-- src/mcp/auth.ts (constant-time bearer validation)
      +-- src/mcp/tools.ts (MCP tool handlers)
      +-- src/kv.ts (read-only KV helpers)
      |
      v
  MCP endpoints (read: get_context, list_sections, get_meta)
                 (write: append_journal, list_journal)
                 |
                 +-- Authentication: two tokens, two scopes (read/write)
                 +-- Journal: append-only, ULID keys, manual promotion gate

Key Design Decisions

  • Hand-curated Markdown, not vector search. Nothing becomes memory without explicit human approval. Avoids the semantic-drift problem of vector-store memory when facts change over time.
  • Append-only journal, manual promotion. Agents can leave notes but cannot corrupt curated context. The promotion step is where the owner filters signal from noise.
  • Two token scopes, separate secrets. Read token for sessions, write token for servers. Write leak is contained (notes only), read leak is the primary risk (context exposure).
  • Constant-time token comparison. No timing attacks. Uses crypto.subtle.digest (SHA-256 + XOR accumulate), portable across workerd and Node.

Quick Start

Deploy to Cloudflare

npm install
cp wrangler.toml.example wrangler.toml
wrangler kv namespace create CONTEXT_KV
wrangler kv namespace create OAUTH_KV
wrangler secret put READ_TOKEN
wrangler secret put WRITE_TOKEN
cp -r content.example content
npm run build
wrangler kv bulk put artifacts/kv-bulk.json --binding CONTEXT_KV
wrangler deploy

Connect Claude Code

claude mcp add --transport http context-kernel \\
  https://my-context-kernel.myname.workers.dev/mcp \\
  --header "Authorization: Bearer <READ_TOKEN>"

The skill .claude/skills/context-kernel/SKILL.md auto-loads your context on every session start.

Append via Agents (write-token)

Agents and servers use WRITE_TOKEN to append journal entries. You review via npm run promote and hand-promote what becomes permanent.

OAuth for claude.ai Chat

Not yet implemented (provider library incompatibility). Claude Code CLI + local servers work today.
